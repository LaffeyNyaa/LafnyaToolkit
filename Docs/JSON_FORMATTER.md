# JSON Formatter

## 概述

JSON 格式化工具位于 `JsonFormatter` 目录。它接受一个目标目录作为参数，递归格式化该目录及其子目录下的所有 `*.json` 文件，遵循 RFC 8259 最佳格式化实践。解析器为手写递归下降实现，不引用任何外部 JSON 库。

### 执行行为

- **无效输入：** 如果指定的目录不存在，或没有指定参数，工具会向 stderr 输出错误提示并以退出码 2 退出，不修改任何文件。
- **进度日志：** 格式化过程中，工具会显示每个文件正在格式化或已跳过的信息。文件内容在格式化前后一致，则算作跳过格式化。
- **汇总报告：** 格式化完毕后，输出汇总行 `Total: N, Formatted: M, Skipped: K, Failed: L`，包含找到的文件总数、格式化的文件数、跳过的文件数、失败的文件数（`N = M + K + L`）。
- **错误处理：** 如果某个文件解析失败（如 JSON 语法错误），工具会向 stderr 输出 `Error: <相对路径>: <错误信息>`（错误信息包含 1-based 行号与列号），计入 `Failed`，不写入文件，继续处理剩余文件。

## 格式化规则

### 缩进

每级缩进使用恰好 **2 个空格**。输出中不使用 tab 字符。

### 对象格式

非空对象输出为多行：
- `{` 在起始行
- 每个键值对独占一行，缩进比起始 `{` 深一级
- 键值对格式为 `"key": value`（冒号后单个空格）
- 除最后一个键值对外，每个键值对后跟逗号
- `}` 在结束行，缩进与起始 `{` 相同

空对象输出为单行 `{}`。

```json
{
  "name": "Lafnya",
  "version": "0.1.0",
  "empty": {}
}
```

### 数组格式

非空数组输出为多行：
- `[` 在起始行
- 每个元素独占一行，缩进比起始 `[` 深一级
- 除最后一个元素外，每个元素后跟逗号
- `]` 在结束行，缩进与起始 `[` 相同

空数组输出为单行 `[]`。

```json
[
  1,
  2,
  3
]
```

### 键顺序保留

对象键按原始输入顺序序列化，不进行字典排序。重复键原样保留（不去重）。

### 标量字面量保留

数字与字符串的原始字面量在输出中逐字保留，不做规范化：
- 数字 `1.0` 保持 `1.0`（不变为 `1`）
- 数字 `1e10` 保持 `1e10`（不展开）
- 数字 `-0` 保持 `-0`
- 字符串转义序列（如 `\"`、`\n`、`\u0041`、`\/`）原样保留，不重新转义或反转义

### 行尾与文件末尾换行

- 所有换行一律使用 LF (`\n`)
- 文件末尾恰好一个 `\n`
- 无行尾空白
- **优先规则：** 与其他规则冲突时，单个尾部换行的要求具有绝对优先权

### 文件编码

- 读取时使用 UTF-8，容忍并跳过开头的单个 UTF-8 BOM
- 写入时使用 UTF-8 无 BOM

### JSON 语法

遵循 RFC 8259：
- 顶层值可以是对象、数组、字符串、数字、`true`、`false`、`null` 中的任意一种
- 仅处理 `*.json` 扩展名（Windows 上不区分大小写）
- 递归发现目标目录及其子目录下的所有 `*.json` 文件，按字母序（OrdinalIgnoreCase）处理

### 退出码

- 参数缺失或路径无效：退出码 2
- 正常处理完毕（无论是否有文件解析失败）：退出码 0
